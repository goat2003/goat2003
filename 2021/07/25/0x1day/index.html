暑期学习笔记 0x1

c++：

int a;

int& x=a; 是一种引用的方法，就是给一个变量起”别名”，并且定义时必须初始化。

使用这种方法可以直接调用变量a的地址，值么一说是不是特别像指针变量啊。

我们直接去看一下反汇编。下图分别是指针和引用

看反汇编


是不是发现一摸一样，所以这种引用法可以理解为简化的指针，那么引用发就真的和指针一摸一样吗？当然不是，在我们使用引用的时候，系统会自动讲引用转换成int* const ref=&a ; 这就使引用指向的地址不可改变。

const int& ref=10; 刚说过引用在定义的时候必须初始化，但是10明显不是一个合法的地址，为什么能直接指向10呢？是因为前面加上了一个const使得系统会自动给我们创建一个临时变量 int temp=10;然后再给我们转换成const int& ref=temp;

所以现在ref就既不能改变指向，也不能改变指向的地址的值了。
c++函数定义时参数可以定义一个默认值，但是我们要是先声明再定义的话，定义的时候不可以加上默认值，为什么呢？假如说我们再声明的时候默认值是10，定义的时候改成了20，系统会分不清按哪个值来执行，自然就会报错，所以定义的时候也赋值默认值的话会无法执行。

函数的占位参数，再定义一个函数时，可以先放一个int 后面不加变量预定义一个参数，但是调用的时候必须加上一个对应的实参，这就是函数的占位参数。
看到这里难免会有疑惑，用这个占位参数的目的是什么，同疑惑，后续会讲，那就后续在解决吧。
